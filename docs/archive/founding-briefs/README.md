# Founding Briefs Archive

These PDFs are **historical steering / handoff documents** from the earliest Prism formation phase.

They remain valuable as origin material, but they are **not the current source of truth** for the project.

## What these files are
- `blueprint.pdf` — early founding product/infrastructure blueprint
- `expansion-strategy.pdf` — short strategic steer memo
- `node-protocol.pdf` — deeper early thinking on node roles, rollout, trust, and resilience

## Status
These documents predate the current Prism markdown documentation and still use some earlier framing (including the pre-Prism naming).

They should be treated as:
- **archival reference**
- useful for project provenance and early intent
- not authoritative for current implementation decisions

## Current source of truth
Use these markdown docs instead when making active product or engineering decisions:
- `README.md`
- `ROADMAP.md`
- `docs/APP_VISION.md`
- `docs/AI_ARCHITECTURE.md`
- `docs/WEBSITE.md`

- `DECISIONS.md`

## Note on future cleanup
If material from these PDFs is still strategically useful, it should be extracted into current markdown docs rather than reactivated in PDF form.
